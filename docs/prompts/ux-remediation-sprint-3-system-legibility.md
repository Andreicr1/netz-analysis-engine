## What you are fixing

This sprint covers **BL-08 through BL-12** — System Legibility items that make AI-generated content auditable and traceable.

### BL-08 — Evidence Pack Inspector: IC Memos

**Problem:** `MemoEvidencePack.evidence_json` (frozen source of truth for IC memo chapter generation) is tracked in the database with full metadata (version_tag, token_count, model_version, is_current) but never serialized to the frontend. Users see generated chapters but cannot inspect the evidence that produced them.

**Files to modify:**
- Backend: IC memo routes (add evidence pack serialization endpoint or extend existing)
- `frontends/credit/src/lib/components/ICMemoViewer.svelte` — add "View Sources" per chapter
- `frontends/credit/src/lib/components/ICMemoStreamingChapter.svelte` — add trigger for evidence drawer

**Research needed before implementation:**
- Read `MemoEvidencePack` model to understand `evidence_json` structure
- Check if existing IC memo endpoints already serialize evidence packs
- Determine if model version is per-memo or per-chapter

**Acceptance criteria:**
- Each IC memo chapter has a "View Sources" action (button or link)
- Action opens `Sheet` component (from `@netz/ui`) showing: source document name, page range, extraction confidence, chunk ID
- Evidence data loads on demand (not included in initial page load unless lightweight)
- Empty evidence packs show "No evidence sources recorded" message

### BL-10 — AI-Generated Content Markers on Memos and Reports

**Problem:** IC memo chapters and DD report sections render as undifferentiated markdown. No inline indicator distinguishes AI-generated narrative from evidence. Backend tracks `model_version`, `generated_at`, `token_count` per chapter.

**Scope:** Frontend only. Add a provenance caption below each chapter/section header.

**Files to modify:**
- `frontends/credit/src/lib/components/ICMemoStreamingChapter.svelte`
- `frontends/wealth/src/routes/(team)/dd-reports/[fundId]/[reportId]/+page.svelte`

**Research needed before implementation:**
- Verify `model_version` and `generated_at` are included in API responses for IC memo chapters and DD report chapters
- Check if `model_version` being non-null is a reliable indicator of AI-generated content

**Acceptance criteria:**
- Each AI-generated chapter/section displays a subdued caption: `"Generated by {model} on {date}"`
- Caption uses `--netz-text-caption` size and `--netz-info` color (calm, not alarming)
- Caption does not appear on human-written or manually edited content
- Caption is consistent across Credit IC memos and Wealth DD reports

**Design note:** Avoid loud "AI WARNING" labels. Use disciplined, calm annotations per the UX doctrine.

**Files to modify:**

- Wealth document list pages (if they exist)

**Research needed before implementation:**

### BL-12 — StatusBadge Unknown State Dev Warning

**Problem:** `StatusBadge` silently renders unrecognized states as neutral gray. No console warning, no dev-mode indicator. New backend states without frontend updates render as uninformative gray badges.

**Scope:** Shared `@netz/ui` component.

**Files to modify:**
- `packages/ui/src/lib/components/StatusBadge.svelte` (line 74 fallback logic)

**Dependencies:** None. Can ship independently.

**Acceptance criteria:**
- `console.warn('StatusBadge: unrecognized status "{token}"')` fires in dev mode for unknown tokens
- Dashed border renders on unknown tokens in dev builds only (no production visual change)
- Known "neutral" statuses (explicitly listed) do not trigger the warning
- Define explicit `NEUTRAL_STATUSES` set so intentionally-neutral states like "inactive" or "archived" don't trigger false positives

## Critical rules (from CLAUDE.md)

- **Frontends never cross-import:** `frontends/credit/` and `frontends/wealth/` share only via `@netz/ui` and the backend API
- **Frontend formatter discipline:** Use formatters from `@netz/ui` (`formatDate`, etc.). Never `.toFixed()` or inline `Intl`
- **Svelte 5 runes:** Use `$state`, `$derived`, `$effect` syntax. Escape `$` as `\$` in terminal
- **SSE:** Use `fetch()` + `ReadableStream` — never `EventSource`
- **Pydantic schemas:** All routes use `response_model=` and return via `model_validate()`
- **Prompts are Netz IP:** Never expose prompt content in client-facing API responses
- Do not add features beyond scope. This sprint is strictly BL-08 through BL-12.
