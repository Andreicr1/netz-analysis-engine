## 2. Migrations (0038–0042)

### 2.1. Migration 0038 — Manager Screener Indexes + Continuous Aggregates

**Arquivo:** `backend/app/core/db/migrations/versions/0038_manager_screener_indexes_continuous_aggs.py`

**Indexes:**

| Tabela | Index | Colunas |
|--------|-------|---------|
| `sec_managers` | `idx_sec_managers_aum` | `aum_total DESC` |
| `sec_managers` | `idx_sec_managers_compliance_aum` | `compliance_disclosures, aum_total DESC` |

**Continuous Aggregates (TimescaleDB, refresh diario):**

**`sec_13f_holdings_agg`** — Holdings por setor por quarter:
```sql
SELECT cik,
  time_bucket('3 months', report_date) AS quarter,
  sector,
  SUM(market_value)        AS sector_value,
  COUNT(DISTINCT cusip)    AS position_count
FROM sec_13f_holdings
WHERE asset_class = 'COM'
GROUP BY cik, time_bucket('3 months', report_date), sector
```

**`sec_13f_drift_agg`** — Churn e mudancas por quarter:
```sql
SELECT cik,
  time_bucket('3 months', quarter_to) AS quarter,
  COUNT(*) FILTER (WHERE action IN ('NEW_POSITION','EXITED')) AS churn_count,
  COUNT(*) AS total_changes
FROM sec_13f_diffs
GROUP BY cik, time_bucket('3 months', quarter_to)
```

**Indexes adicionais:**
- `idx_sec_13f_holdings_agg_cik_quarter` — `(cik, quarter DESC)`
- `idx_sec_13f_drift_agg_cik_quarter` — `(cik, quarter DESC)`

### 4.2. SEC — N-PORT Service

**Arquivo:** `backend/data_providers/sec/nport_service.py`

| Metodo | API Externa | Retorno | Destino DB |
|--------|-------------|---------|------------|
| `fetch_holdings(cik, months=12)` | EDGAR N-PORT XML (edgartools) | `list[NportHolding]` | `sec_nport_holdings` |

**Fluxo:**
1. Verifica staleness no DB (retorna cache se < `staleness_ttl_days`)
2. `Company(cik).get_filings(form="NPORT-P")` via edgartools
3. Parse XML iterativo (`invstOrSec` elements) — namespace-agnostic
4. Extrai: cusip, isin, name, assetCat, issuerCat, valUSD, balance, curCd, pctVal, isRestrictedSec, fairValLevel
5. Upsert em chunks de 2000 rows
6. Re-read do DB → frozen dataclasses

**Limites:** 8 req/s (EDGAR rate limit), max 20K holdings por filing, upserts em batches de 2000.

### 5.1. SEC Refresh Worker (#205)

**Arquivo:** `backend/app/domains/wealth/workers/sec_refresh.py`
**Lock ID:** 900_016 | **Escopo:** global

**Fluxo:**
1. Refresh continuous aggregates: `sec_13f_holdings_agg`, `sec_13f_drift_agg`
2. Computa aggregados por manager em bulk:
   - Latest quarter holdings → `portfolio_value`, `position_count`, `top_sector`, HHI
   - Drift → `turnover`, `style_drift_detected`
3. Escreve Redis cache: `screener:agg:{crd_number}` (TTL 24h com jitter ±1h)

**Retorno:** `{status, managers_processed, holdings_aggregated, drift_aggregated, redis_keys_written}`

### 5.2. N-PORT Ingestion Worker (#211)

## 10. Macro Enrichment Integration

**Arquivo:** `backend/quant_engine/regional_macro_service.py`
**Todo:** #216

### Funcoes Adicionadas

#### `score_credit_cycle(region, bis_data)` → `CreditCycleScore`

7a dimensao do scoring regional, composta por dados BIS:

| Sub-indicador | Peso | Logica |
|---------------|------|--------|
| Credit-to-GDP gap | 50% | Invertido: gap positivo = overheating → score baixo |
| Debt service ratio | 30% | Invertido: DSR alto = stress → score baixo |
| Property prices | 20% | Nao-linear: crescimento moderado = saudavel, >5% = overheating |

**Mapeamento:** Paises → regioes via `_COUNTRY_TO_REGION` (44 paises → US/EUROPE/ASIA/EM).

#### `blend_imf_growth(region, fred_growth_score, imf_data, as_of)` → `float`

Blenda score de crescimento FRED (backward-looking) com forecast IMF (forward-looking):

| Componente | Peso | Fonte |
|------------|------|-------|
| FRED GDP growth score | 70% | `macro_data` hypertable (via worker existente) |
| IMF GDP forecast score | 30% | `imf_weo_forecasts` hypertable (via IMF worker) |

Usa forecasts do ano corrente e proximo (`NGDP_RPCH`). Converte media GDP → score 0-100.

#### `enrich_region_score(result, bis_data, imf_data)` → `RegionalMacroResult`

Funcao pura (sem I/O). Recebe `RegionalMacroResult` existente + dados BIS/IMF pre-carregados.

1. Adiciona dimensao `credit_cycle` (peso 0.10 no composite)
2. Blenda score `growth` com forecast IMF
3. Recomputa `composite_score` com pesos atualizados
4. Retorna novo `RegionalMacroResult` imutavel

**Safe defaults:** Sem dados BIS/IMF → retorna resultado original inalterado.

## 13. Decisoes Tecnicas

| Decisao | Razao |
|---------|-------|
| BIS via CSV (nao SDMX/sdmx1) | Evita dependencia pesada; BIS oferece CSV nativo |
| IMF via JSON simples (nao SDMX) | DataMapper API retorna JSON direto, sem parsing complexo |
| PyMuPDF para brochures (nao OCR/Mistral) | SEC exige PDFs text-searchable desde 2010; texto ja embutido |
| Continuous aggregates (nao views) | Pre-computado, refresh automatico, instant reads para screener |
| Redis cache com jitter ±1h | Previne thundering herd no refresh de cache |
| ISO-2 como padrao DB | Consistencia entre BIS (nativo ISO-2) e IMF (convertido de ISO-3) |
| Advisory locks deterministicos | IDs fixos (900_014–900_018), nunca `hash()` |
| Upserts em chunks de 2000 | Limite de parametros asyncpg |
| `compress_segmentby = 'country_code'` | Otimiza queries por pais (BIS/IMF sempre filtram por country) |
| Funcoes puras no regional_macro_service | Zero I/O, thread-safe, testavel isoladamente |
| 7a dimensao `credit_cycle` peso 0.10 | Nao domina o composite; complementa as 6 dimensoes FRED existentes |
| Blend FRED 70% + IMF 30% | FRED = dados realizados (mais confiaveis), IMF = projecao (complementar) |
