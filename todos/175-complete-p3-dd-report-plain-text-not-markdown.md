# DD Report Chapters Rendered as Plain Text Instead of Markdown

## Problem Statement

DD report chapters are generated by the backend as Markdown content but the frontend renders them using plain text interpolation, losing all formatting (headings, bold, lists, tables, links).

## Findings

- **File:** `frontends/wealth/src/routes/(team)/dd-reports/[fundId]/[reportId]/+page.svelte` line 125
- Uses `{chapters[activeChapter].content}` (Svelte text interpolation) instead of a Markdown renderer
- The DD report engine (`vertical_engines/wealth/dd_report_engine.py`) produces chapter content as structured Markdown with headings, bullet lists, and bold emphasis
- UX specification calls for a "sanitizing Markdown renderer (marked + DOMPurify)" to prevent XSS while preserving formatting
- All chapter content displays as a wall of unformatted text, significantly degrading readability

## Proposed Solution

1. Add a `renderSafeMarkdown` utility to `frontends/wealth/src/lib/utils/` that uses `marked` for parsing and `DOMPurify` for sanitization
2. Replace `{chapters[activeChapter].content}` with `{@html renderSafeMarkdown(chapters[activeChapter].content)}`
3. Add prose typography styles (Tailwind `prose` class) to the content container
4. Install `marked` and `dompurify` as dependencies (with `@types/dompurify` for TypeScript)

## Acceptance Criteria

- [ ] DD report chapters render Markdown formatting (headings, lists, bold, links, tables)
- [ ] Output is sanitized — no raw HTML injection possible from chapter content
- [ ] `renderSafeMarkdown` utility is reusable across other pages that display generated content
- [ ] No visual regression on chapters that contain minimal formatting
